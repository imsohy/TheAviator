# 월드 좌표계 축 표시 기능

본 문서는 **메인 게임 화면**(`index.html` → `src/main.js` → `src/game.js`)에서 월드 기준 XYZ 방향을 확인하기 위해 추가한 **좌표축 표시**만 정리합니다. 색감·렌더러 출력 색 공간 등 다른 설정은 이 기능과 무관하며, **색감용 코드는 변경하지 않았습니다.**

---

## 목적

- 씬의 **월드 좌표계**에서 X, Y, Z가 각각 어느 방향인지 한눈에 보이게 함.
- 디버깅·강의·과제 설명용 보조 시각화.

---

## 구현 요약

| 항목 | 내용 |
|------|------|
| API | Three.js `AxesHelper` |
| 적용 파일 | `src/game.js` |
| 삽입 위치 | `createScene()` 내부, `scene` 생성 직후 |
| 축 길이 | 생성자 인자 **300** (월드 단위) |

---

## 코드 위치

`createScene()`에서 씬을 만든 뒤 아래 두 줄이 추가됩니다.

```javascript
const worldAxes = new THREE.AxesHelper(300);
scene.add(worldAxes);
```

---

## 축 색상 규칙 (Three.js 기본)

`AxesHelper`는 Three.js 관례에 따라 다음과 같습니다.

| 색 | 축 | 방향 |
|----|-----|------|
| 빨강 | **X** | 월드 +X |
| 초록 | **Y** | 월드 +Y |
| 파랑 | **Z** | 월드 +Z |

원점 `(0, 0, 0)`에서 각 축의 **양의 방향**으로 선이 그려집니다.

---

## 동작 범위

- **메인 게임**에만 적용되어 있습니다. (`part1.js`, `part2.js`에는 동일 축을 넣지 않았습니다.)
- 축은 `scene`에 직접 붙어 있어 **월드 좌표계 기준**으로 고정됩니다. 비행기·카메라가 움직여도 축의 의미는 변하지 않습니다.

---

## 조정 시 참고

- **길이 변경:** `new THREE.AxesHelper(300)`의 숫자만 바꾸면 됩니다. (작게 하면 화면 가림이 줄어듭니다.)
- **끄기:** 위 두 줄을 주석 처리하거나 삭제하면 됩니다.
- Part 1·2에도 동일하게 보이게 하려면 각 파일의 `createScene()`에 같은 패턴을 추가하면 됩니다.

---

## 요약

- 월드 기준 XYZ 방향을 **빨강·초록·파랑** 선으로 표시하는 기능입니다.
- **색감(렌더러 출력 색 공간, 조명, 안개, 재질 색 등)은 이 문서의 기능 추가로 수정하지 않았습니다.**
